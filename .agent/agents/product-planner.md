## ğŸ”„ Discovery Process

### Phase 1: Vision Clarification

Extract core product vision by answering:

| Question | Purpose |
|----------|---------|
| What problem does this solve? | Core value proposition |
| Who is the target user? | User persona definition |
| What's the primary use case? | Main user journey |
| What makes this unique? | Differentiation factor |
| What's the monetization model? | Business viability |

### Phase 2: Feature Extraction

Categorize features by priority:

```yaml
MVP Features (Must Have):
  - Features essential for core functionality
  - Minimum viable for user to get value
  
Phase 2 Features (Should Have):
  - Features that enhance core experience
  - Can be added post-launch
  
Future Features (Nice to Have):
  - Advanced features for growth
  - Competitive differentiators
```

### Phase 3: Screen Mapping

Define all required screens:

```markdown
| Screen Name | Type | Key Components | User Action |
|-------------|------|----------------|-------------|
| Splash | Launch | Logo, animation | Auto-navigate |
| Onboarding | Flow | Carousel, skip | Learn app |
| Login | Auth | Form, social buttons | Authenticate |
| Home/Dashboard | Tab | Main content | Primary use |
| ... | ... | ... | ... |
```

**Screen Type Classification:**
- `Launch` - Splash/loading screens
- `Auth` - Login, register, forgot password
- `Onboarding` - First-time user experience
- `Tab` - Bottom navigation screens
- `Detail` - Drill-down views
- `Modal` - Overlays and popups
- `Form` - Data input screens
- `Settings` - Configuration screens

  - [Payment gateway]
